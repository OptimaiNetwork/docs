# Governance & Tokenomics

This page separates what is defined today from what should be finalized before public tokenomics publication.

## Current OPI Utility

OPI is the coordination asset for OptimAI Network. It is intended to support:

- node and validator rewards
- data campaign budgets
- Claw extraction jobs
- Search and API usage
- Persona Agent services
- marketplace payments
- staking and reputation
- governance participation

## Economic Design Principles

### Reward useful work

Rewards should favor validated contribution, not raw activity. A node that submits accurate, fresh, source-backed data should be weighted above a node that submits more volume with lower quality.

### Make demand visible

Campaigns, API usage, Claw jobs, and marketplace orders should make demand measurable. This helps the network route work toward what users and builders actually need.

### Align long-term participants

Staking, reputation, and governance should reward reliability and discourage low-quality or abusive behavior.

### Keep tokenomics auditable

Public tokenomics should include supply, allocation, unlocks, reward pools, treasury usage, and governance rights when finalized.

## Governance Direction

OptimAI governance is expected to evolve in phases:

| Phase | Description |
| --- | --- |
| **Foundation** | Core team defines protocol parameters, rewards, and safety policies. |
| **Community signal** | OPI holders and contributors provide feedback on incentives, campaigns, and ecosystem priorities. |
| **Governance alpha** | On-chain voting begins for selected parameters or working groups. |
| **DAO maturity** | Broader governance over grants, marketplace policy, protocol upgrades, and treasury allocation. |

## Decisions Governance May Cover

- reward formulas and campaign incentives
- validator requirements
- supported node task categories
- data policy and source restrictions
- ecosystem grants
- marketplace rules
- protocol upgrades
- treasury spending

## Tokenomics Fields To Finalize

The docs should not invent token numbers. Before publishing detailed tokenomics, confirm:

- total supply
- initial circulating supply
- ecosystem reward allocation
- team and advisor allocation
- treasury allocation
- investor allocation, if applicable
- vesting and unlock schedule
- staking rewards
- governance rights
- chain, bridge, and contract addresses
