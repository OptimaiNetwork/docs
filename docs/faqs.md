# FAQs

Common questions about OptimAI Network, products, nodes, privacy, and OPI.

## General

<details>
<summary>What is OptimAI Network?</summary>

OptimAI Network is a decentralized intelligence network for agentic AI. It combines browser-native nodes, reinforcement data pipelines, DePIN compute, human validation, OptimAI Chain, and OPI incentives to power products such as OptimAI Search, OptimAI Claw, and OptimAI Persona Agent.
</details>

<details>
<summary>What is the main vision?</summary>

OptimAI aims to become the intelligence layer for the agentic internet: a network where people contribute useful data, validation, compute, and feedback, then share in the value created by AI products and agents.
</details>

<details>
<summary>How is OptimAI different from a normal AI app?</summary>

OptimAI is both a network and a product ecosystem. The network supplies fresh data, provenance, validation, compute, and rewards. The product layer turns that infrastructure into Search, Claw, Persona Agent, APIs, and future agent applications.
</details>

## Products

<details>
<summary>What is OptimAI Search?</summary>

OptimAI Search is a Web3 AI search engine powered by OptimAI Network. It retrieves and verifies sources from web and social contexts, then synthesizes answers with citations and visual context when available.
</details>

<details>
<summary>What is OptimAI Claw?</summary>

OptimAI Claw is the native agent runtime inside OptimAI Core Node. It can plan tasks, search for context, extract structured data, summarize findings, and execute workflows from user instructions.
</details>

<details>
<summary>What is OptimAI Persona Agent?</summary>

Persona Agent is the personal AI layer of OptimAI. It learns from permissioned user context, remembers preferences and workflows, uses Search and Claw as tools, and helps users complete personalized tasks.
</details>

<details>
<summary>Do I need a wallet to use OptimAI products?</summary>

Some product experiences may work without a wallet. Wallet connection can unlock Web3-native features such as rewards, OPI utility, staking, marketplace activity, or advanced account ownership.
</details>

## Nodes

<details>
<summary>What are OptimAI Nodes?</summary>

OptimAI Nodes are the participation layer of the network. They help provide data, validation, bandwidth, compute, storage, edge context, and browser-native task execution.
</details>

<details>
<summary>What is the difference between Lite, Core, and Edge Nodes?</summary>

Lite Nodes are lightweight and easy to use through browser extensions or Telegram. Core Nodes are desktop or CLI nodes for heavier data, browser, compute, and campaign tasks. Edge Nodes run on mobile and future IoT environments for local and context-rich participation.
</details>

<details>
<summary>Can I run OptimAI Nodes on multiple devices?</summary>

Yes. Users can run nodes on multiple supported devices as long as they follow the network’s account, task, and reward rules.
</details>

<details>
<summary>Do I need technical knowledge to run a node?</summary>

Lite Node is designed for non-technical users. Core Node and CLI workflows are better suited for users comfortable with desktop apps, Docker, or command-line tools.
</details>

## Data And Privacy

<details>
<summary>What kind of data does OptimAI collect?</summary>

Depending on node type and user permissions, OptimAI can process public web data, social or community signals, extraction campaign data, validation responses, document content, browsing workflow context, and edge signals. Sensitive or permissioned data should always follow consent and privacy controls.
</details>

<details>
<summary>How does OptimAI protect privacy?</summary>

OptimAI is designed around user control, local processing where possible, anonymization, encryption, configurable participation, and permissioned access for private contexts.
</details>

<details>
<summary>Can I control what I share?</summary>

Yes. OptimAI participation is intended to be configurable. Users should be able to choose task types, pause participation, and control permissioned data access as product features evolve.
</details>

<details>
<summary>Why does OptimAI use browser-native nodes?</summary>

Many valuable AI signals live inside modern browser workflows: dynamic pages, logged-in tools, dashboards, social feeds, and multi-step user journeys. Browser-native nodes can understand these environments better than traditional crawlers, while preserving user consent and control.
</details>

## OPI And Rewards

<details>
<summary>What is OPI?</summary>

OPI is the utility and coordination token of OptimAI Network. It supports rewards, service fees, campaign payments, staking, reputation, marketplace transactions, and governance.
</details>

<details>
<summary>How are rewards calculated?</summary>

Rewards are expected to consider useful contribution, task type, data quality, validation consistency, node reliability, campaign demand, compute or bandwidth contribution, and reputation.
</details>

<details>
<summary>What can I use OPI for?</summary>

OPI can support OptimAI services such as Search, Claw, Persona Agent, APIs, data campaigns, marketplace services, compute usage, staking, and governance.
</details>

## Builders

<details>
<summary>Can developers build on OptimAI?</summary>

Yes. OptimAI is moving toward APIs and SDKs for search, extraction, data campaigns, agent workflows, marketplace services, and compute-backed applications.
</details>

<details>
<summary>What can builders create?</summary>

Builders can create research tools, monitoring agents, data products, vertical AI apps, Persona extensions, Claw workflows, dashboards, datasets, and marketplace services.
</details>
