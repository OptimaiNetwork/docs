# Data Schemas

:::info[Preview schema]
These schemas define the recommended shape for OptimAI data objects. They are intended for product, API, and integration alignment.
:::

Professional AI docs need predictable objects. The following schemas describe the key entities builders will work with across Search, Claw, Persona Agent, Campaigns, Nodes, and Rewards.

  "id": "src_01JZ9K9B2MYYD2QJKF1EVW93X8",
  "type": "web_page",
  "url": "https://docs.optimai.network/",
  "title": "OptimAI Network Docs",
  "captured_at": "2026-06-22T08:20:00Z",
  "freshness": "current",
  "metadata": {
    "domain": "docs.optimai.network",
    "language": "en",
    "content_type": "documentation"

## Schema Design Rules

- Use stable IDs with prefixes such as `src_`, `claw_`, `mem_`, `camp_`, `node_`, and `rew_`.
- Include `created_at` or `captured_at` timestamps for auditability.
- Preserve provenance for all data used by agents.
- Separate raw source metadata from validated quality signals.
- Keep private Persona memory scoped to the user unless explicitly shared.
